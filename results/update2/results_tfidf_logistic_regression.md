# TF-IDF + Logistic Regression Baseline — Results (HaluEval QA)

## Experiment Overview

**Task:** Binary hallucination detection — classify whether a given answer to a question is factually consistent with the provided knowledge, or hallucinated.

**Dataset:** [HaluEval](https://arxiv.org/abs/2305.11747) QA subset (`pminervini/HaluEval` on Hugging Face)

**Model:** TF-IDF (50,000 features, unigrams + bigrams) + Logistic Regression (`saga` solver, C=1.0)

**Notebook:** `notebooks/02b_tfidf_baseline_LogisticRegression.ipynb`

---

## Dataset Summary

| Property | Value |
|---|---|
| Source | HaluEval QA (HotpotQA-based) |
| Raw samples | 10,000 QA pairs |
| Expanded samples | 20,000 (each pair → 1 correct + 1 hallucinated row) |
| Training set | 16,000 (80%) |
| Validation set | 4,000 (20%) |
| Classes | `Not Hallucinated`, `Hallucinated` |
| Class balance | Perfectly balanced (50/50) |

Each sample consists of:
- **Knowledge:** Wikipedia evidence passage
- **Question:** A factual question
- **Answer:** Either the ground-truth answer (from HotpotQA) or a ChatGPT-generated hallucinated answer

The text input to the model is the concatenation: `"Knowledge: {knowledge} Question: {question} Answer: {answer}"`.

## Analysis

### Why 21% Accuracy? (Below Random Chance)

The baseline achieves **21% accuracy on a balanced binary task**, which is significantly below the 50% random-chance baseline. This is a meaningful and informative negative result:

1. **Anti-correlated surface features:** The TF-IDF model picks up lexical cues that are *inversely* correlated with the true labels. Hallucinated answers (generated by ChatGPT) tend to be more elaborate, fluent, and stylistically different from the concise ground-truth answers sourced from HotpotQA. The model learns to associate these style patterns with the wrong class.

2. **Shared context dominates:** For each QA pair, the knowledge passage and question are identical between the hallucinated and non-hallucinated versions. Since TF-IDF treats the entire concatenated text as a bag of words, the discriminative signal (the answer portion) is drowned out by the shared knowledge and question tokens.

3. **Hallucination detection requires semantic reasoning:** Detecting whether an answer is consistent with a knowledge passage requires *entailment-level reasoning* — understanding whether the answer's claims are supported by the evidence. Bag-of-words features fundamentally cannot capture this.

### Key Takeaways

- TF-IDF on the full concatenated text + Logistic Regression is **not suitable** for hallucination detection on HaluEval.
- The model's worse-than-random performance confirms that **lexical overlap is anti-predictive** for this task — hallucinated answers can be lexically similar to evidence while being factually wrong.
- This baseline establishes a clear **lower bound** and motivates the need for better feature engineering or semantic approaches.

## Comparison with Improved Baseline

When the feature engineering is fixed (answer-only TF-IDF + handcrafted overlap features), the same Logistic Regression model achieves **96.13% accuracy** — see `RESULTS.md` and `notebooks/02_baseline_tfidf_xgboost.ipynb` for details.

| Pipeline | Accuracy |
|---|---|
| TF-IDF (full text) + LR **(this file)** | 0.2095 |
| TF-IDF (answer-only) + overlap features + LR | 0.9613 |
| TF-IDF (answer-only) + overlap features + Gradient Boosting | 0.9615 |
| Fine-tuned DeBERTa-v3-base (see `01_baseline_nli_finetuning.ipynb`) | ~0.91 |
