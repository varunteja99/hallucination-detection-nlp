# Hallucination Detection Baseline — Results (HaluEval QA)

## Experiment Overview

**Task:** Binary hallucination detection — classify whether a given answer to a question is factually consistent with the provided knowledge, or hallucinated.

**Dataset:** [HaluEval](https://arxiv.org/abs/2305.11747) QA subset (`pminervini/HaluEval` on Hugging Face)

**Model (Improved):** Answer-only TF-IDF (30,000 features, unigrams–trigrams) + handcrafted overlap features + Gradient Boosting classifier

**Previous baseline:** TF-IDF on full concatenated text + Logistic Regression → 21% accuracy (below random chance)

---

## Dataset Summary

| Property | Value |
|---|---|
| Source | HaluEval QA (HotpotQA-based) |
| Raw samples | 10,000 QA pairs |
| Expanded samples | 20,000 (each pair → 1 correct + 1 hallucinated row) |
| Training set | 16,000 (80%) |
| Validation set | 4,000 (20%) |
| Classes | `Not Hallucinated`, `Hallucinated` |
| Class balance | Perfectly balanced (50/50) |

---

## What Changed (21% → 96.15%)

| Change | Why It Matters |
|---|---|
| **Answer-only TF-IDF** | Removed shared knowledge/question tokens that drowned the discriminative signal |
| **Handcrafted overlap features** | Jaccard similarity, unigram precision/recall, length ratios capture the knowledge–answer *relationship* |
| **Trigram TF-IDF** | Captures phrase-level patterns (e.g., "was created by" vs "was invented by") |
| **Multiple classifiers** | Tested 6 models; Gradient Boosting narrowly beat Logistic Regression |

## Analysis

### Why 96% Accuracy Now? (Up from 21%)

The original baseline achieved **21% accuracy** — worse than random chance — because TF-IDF on the full concatenated text (knowledge + question + answer) was dominated by shared tokens. The answer (the only discriminative part) was buried.

The improved pipeline fixes this with two key changes:

1. **Answer-only TF-IDF**: By vectorizing only the answer text, the model focuses on the discriminative signal. Hallucinated answers (generated by ChatGPT) have distinct stylistic patterns — they tend to be more elaborate, use more hedging language, and repeat question context unnecessarily.

2. **Knowledge–answer overlap features**: Handcrafted features (Jaccard similarity, unigram precision/recall between knowledge and answer, length ratios) directly measure how well the answer aligns with the evidence. Factual answers have higher overlap with the knowledge passage.

### Remaining Limitations

- The model relies on **surface-level lexical patterns** rather than true semantic understanding
- Performance may not generalize to domains outside HotpotQA-style questions
- Adversarial hallucinations designed to mimic the style of correct answers would likely fool this model
- A transformer-based approach (see `01_baseline_nli_finetuning.ipynb`, ~91% with DeBERTa) uses semantic reasoning and may be more robust
- TF-IDF + Logistic Regression is **not suitable** for hallucination detection on HaluEval.
- The model's worse-than-random performance confirms that **lexical overlap is anti-predictive** for this task — hallucinated answers can be lexically similar to evidence while being factually wrong.
- This baseline establishes a clear **lower bound** and motivates the need for semantic approaches.
